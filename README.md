# Azure Data Engineering Pipeline for Amazon Sales Data

## Project Overview

This project demonstrates an end-to-end Azure Data Engineering pipeline built using Azure Data Factory, Azure Data Lake Storage Gen2, Azure Databricks (PySpark), and Power BI.

The project follows the Medallion Architecture:

Bronze → Silver → Gold

Raw Amazon sales data is ingested into Azure Data Lake Storage, transformed using Azure Data Factory Mapping Data Flows, processed with Databricks PySpark, and stored as business-ready datasets for analytics and reporting.

## Bronze Layer

Purpose:
Store raw source data without modifications.

Dataset:
Amazon Sales CSV

Storage:
ADLS Gen2 Bronze Container

Operations:
- Raw data ingestion
- No transformations

---

## Silver Layer

Purpose:
Data cleansing and standardization.

Implemented using:
Azure Data Factory Mapping Data Flow

Transformations:
- Column standardization
- Text normalization using upper()
- Null handling
- Replaced missing values in fulfilled-by with UNKNOWN

Result:
Clean and consistent dataset ready for business processing.

---

## Gold Layer

Purpose:
Business-ready analytical datasets.

Implemented using:
Azure Databricks + PySpark

Generated datasets:

### Sales by Category

Business analysis of revenue generated by product category.

### Sales by State

Revenue distribution across states.

### Order Status Summary

Order volume by status.

### Fulfillment Summary

Comparison of Easy Ship and UNKNOWN fulfillment types.

### Top Products

Highest revenue generating products.
